# 02 — Data Model (the centerpiece)

> The requirement stresses: *"Your schema should model the complete domain, including entities
> and relationships you may not have time to fully implement."* This schema goes beyond MVP:
> versioned assets, publishing splits, a reusable artist catalog, and per-artist pitch outcomes.

## ER overview (text)
```
User ─< Member >─ Organization
User ─1:N─ Song (uploadedBy)
Organization ─1:N─ Song
Song ─1:N─ SongAsset            (multiple files / versions)
Song ─1:N─ SongCollaborator ─N:1─ User   (publishing splits / co-writers)
Song ─1:N─ Pitch ─N:1─ User (createdBy / manager)
Pitch ─N:M─ Tag (via PitchTag)
Pitch ─1:N─ PitchTarget ─N:1─ Artist     (per-artist pitch status)
```

## better-auth managed models
`User` (extended with `role`), `Session`, `Account`, `Verification`, `Organization`, `Member`,
`Invitation` — generated by the better-auth Prisma adapter / CLI (`npx @better-auth/cli
generate`), then committed to `schema.prisma`. We extend `User` with a `role` field via
better-auth `additionalFields`. **These tables are written only through better-auth's server
API** — our repositories read them but never insert/update/delete them directly.

## Domain models (Prisma sketch)
```prisma
enum UserRole         { MANAGER SONGWRITER }
enum SongStatus       { DRAFT READY ARCHIVED }
enum PitchStatus      { DRAFT SENT ACCEPTED REJECTED }
enum PitchTargetStatus{ PENDING INTERESTED PASSED CUT }
enum SongAssetKind    { DEMO STEM MIX }
enum CollaboratorRole { WRITER PRODUCER TOPLINE }

model Song {
  id             String   @id @default(cuid())
  title          String
  primaryArtist  String?              // intended/performing artist on the demo
  durationSec    Int?
  bpm            Int?
  musicalKey     String?
  genre          String?
  lyrics         String?  @db.Text
  status         SongStatus @default(DRAFT)
  organizationId String
  uploadedById   String
  organization   Organization @relation(fields:[organizationId], references:[id], onDelete: Cascade)
  uploadedBy     User         @relation(fields:[uploadedById], references:[id])
  assets         SongAsset[]
  collaborators  SongCollaborator[]
  pitches        Pitch[]
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
  @@index([organizationId, status])
  @@index([genre])
  @@index([uploadedById])
}

model SongAsset {
  id        String   @id @default(cuid())
  songId    String
  storageKey String                 // opaque key into StorageService, never a client path
  mimeType  String
  sizeBytes Int
  kind      SongAssetKind @default(DEMO)
  version   Int      @default(1)
  song      Song     @relation(fields:[songId], references:[id], onDelete: Cascade)
  createdAt DateTime @default(now())
  @@index([songId])
}

model SongCollaborator {
  id           String @id @default(cuid())
  songId       String
  userId       String
  splitPercent Decimal @db.Decimal(5,2)   // publishing split
  roleOnSong   CollaboratorRole @default(WRITER)
  song         Song @relation(fields:[songId], references:[id], onDelete: Cascade)
  user         User @relation(fields:[userId], references:[id])
  @@unique([songId, userId])
}

model Pitch {
  id          String @id @default(cuid())
  songId      String
  createdById String
  description String @db.Text
  status      PitchStatus @default(DRAFT)
  song        Song @relation(fields:[songId], references:[id], onDelete: Cascade)
  createdBy   User @relation(fields:[createdById], references:[id])
  tags        PitchTag[]
  targets     PitchTarget[]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  @@index([songId])
  @@index([status])
}

model Tag {
  id     String @id @default(cuid())
  name   String @unique
  pitches PitchTag[]
}

model PitchTag {
  pitchId String
  tagId   String
  pitch   Pitch @relation(fields:[pitchId], references:[id], onDelete: Cascade)
  tag     Tag   @relation(fields:[tagId], references:[id])
  @@id([pitchId, tagId])
}

model Artist {
  id       String @id @default(cuid())
  name     String
  metadata Json?
  targets  PitchTarget[]
}

model PitchTarget {
  id       String @id @default(cuid())
  pitchId  String
  artistId String
  status   PitchTargetStatus @default(PENDING)
  note     String? @db.Text
  pitch    Pitch  @relation(fields:[pitchId], references:[id], onDelete: Cascade)
  artist   Artist @relation(fields:[artistId], references:[id])
  @@unique([pitchId, artistId])
}
```

## Notes
- **Cascade deletes:** deleting a `Song` cascades to its assets, collaborators, pitches; deleting
  a `Pitch` cascades to its `PitchTag`/`PitchTarget` join rows.
- **Indexes** chosen to back the `GET /songs` filters (`organizationId`, `status`, `genre`,
  `uploadedById`) and pitch lookups (`songId`, `status`).
- **`@db.Text`** for free-form long fields (`lyrics`, `description`, `note`).
- **`Decimal(5,2)`** for split percentages to avoid float drift.
